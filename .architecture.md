# Graph View Visualization Architecture

Exhaustive documentation of how the developer tools graph view generates and
renders its graph, contrasted with how the packer uses the same shared modules
for grouping.

## 1. Data Source

Both the graph view and the packer consume `Transition[]` — raw transition
records logged by the bundler whenever the active source changes.

```typescript
// src/aggregation/types.ts:24-30
type Transition = {
    from: string;      // source being left
    to: string;        // source being entered
    ts: number;        // timestamp of the transition
    dwellMs: number;   // time spent in `from` before leaving
};
```

### Where transitions come from

The bundler (`src/aggregation/bundler.ts:51-73`) creates a `Transition` on
every `transition(to)` call. `dwellMs` is derived from the open bundle's
duration. The aggregator (`src/aggregation/index.ts`) calls
`bundler.transition()` whenever `setActiveTab()` fires or a capture arrives
from a different source.

### How the graph view obtains transitions

The graph view reads transitions from `DevEntry[]` props. It scans backwards
for the latest `state.snapshot` event (`GraphView.tsx:230-250`):

```typescript
// GraphView.tsx:230-250
const latestSnapshot = useMemo(() => {
    for (let i = entries.length - 1; i >= 0; i--) {
        const entry = entries[i];
        if (entry.event === "state.snapshot" && entry.data) {
            const snapshot = entry.data as {
                transitions?: Transition[];
                sourceUrls?: Record<string, string>;
            };
            return {
                transitions: collapseShortDwells(snapshot.transitions ?? []),
                sourceUrls: snapshot.sourceUrls ?? {},
            };
        }
    }
    return { transitions: [], sourceUrls: {} };
}, [entries]);
```

**Key detail**: The transitions are immediately passed through
`collapseShortDwells()` before anything else touches them. This is a
graph-view-only preprocessing step that the packer does NOT perform.

## 2. Graph-View-Only Preprocessing: `collapseShortDwells()`

**Location**: `GraphView.tsx:132-155`

This function is unique to the graph view. It collapses short-lived midpoints
out of the transition sequence before the graph is built. The packer does not
use this function.

```typescript
// GraphView.tsx:132-155
const MIN_DWELL_MS = 5000;

function collapseShortDwells(transitions: Transition[]): Transition[] {
    const result: Transition[] = [];
    for (const t of transitions) {
        const prev = result.length > 0 ? result[result.length - 1] : null;
        const connected = prev && prev.to === t.from;
        if (connected && t.dwellMs < MIN_DWELL_MS) {
            // Short-lived midpoint — collapse: rewrite prev.to to t.to
            if (prev.from !== t.to) prev.to = t.to;
        } else if (
            connected &&
            t.from === "off_browser" &&
            t.ts - prev.ts < MIN_DWELL_MS
        ) {
            // Brief off_browser hop during tab switch — dwellMs is unreliable
            // for off_browser (no bundle opened), so use timestamp gap instead
            if (prev.from !== t.to) prev.to = t.to;
        } else {
            result.push({ ...t });
        }
    }
    return result;
}
```

### Behavior

- Scans transitions sequentially, maintaining a result stack.
- If the current transition is connected to the previous one (`prev.to === t.from`)
  and the dwell time is under 5 seconds, it **collapses** the midpoint by
  rewriting `prev.to` to `t.to`. This removes the intermediate source entirely.
- Special case: `off_browser` hops that have no reliable `dwellMs` (no bundle
  is opened for `off_browser`) use a timestamp gap check (`t.ts - prev.ts`)
  instead of `dwellMs`.
- Self-loops (`prev.from === t.to`) are silently dropped to avoid degenerate edges.

### Purpose

Filters out sources that the user flicked through too quickly to represent
meaningful activity — rapid tab switches, accidental focus changes, transient
popups. This produces a cleaner graph for visual inspection than the raw
transition log would.

## 4. Raw Mode Data Pipeline

**Location**: `GraphView.tsx:298-420` (`processEntries` callback)

When `activeTab === "raw"`, the graph is built by directly aggregating the
collapsed transitions:

### Step 1: Find latest snapshot (`GraphView.tsx:312-331`)

Scans `entries[processedRef.current .. entries.length]` for the latest
`state.snapshot` event. Applies `collapseShortDwells()` to its transitions.

### Step 2: Aggregate into edges (`GraphView.tsx:337-350`)

```typescript
// GraphView.tsx:337-350
const newEdges = new Map<string, Edge>();
const nodeIds = new Set<string>();
for (const t of transitions) {
    nodeIds.add(t.from);
    nodeIds.add(t.to);
    const key = `${t.from}->${t.to}`;
    const existing = newEdges.get(key);
    if (existing) {
        existing.weight++;
    } else {
        newEdges.set(key, { from: t.from, to: t.to, weight: 1 });
    }
}
```

Edge keys use `->` separator (unlike the `\0` separator in `buildDirectedGraph`).
Weights are simple transition counts.

### Step 3: Compute first/last seen timestamps (`GraphView.tsx:352-364`)

For each node, scans all transitions to find earliest and latest `ts` values.
These are used for tooltip display.

### Step 4: Update node positions (`GraphView.tsx:366-420`)

- Removes nodes no longer in the transition set.
- Adds new nodes: positioned near an existing neighbor if one is found in the
  transitions, otherwise random within 200px of origin.
- Preserves positions of existing nodes (incremental update).
- Attaches `sourceUrls` to each node.

### Important: No preprocessing, no community detection

Raw mode does NOT call `preprocess()`, `buildDirectedGraph()`, or
`directedLouvain()`. It works directly from `collapseShortDwells()` output.

## 5. Grouped Mode Data Pipeline

**Location**: `GraphView.tsx:261-292` (`groupedResult` useMemo) +
`GraphView.tsx:422-477` (useEffect that populates nodes/edges)

### Step 1: `collapseShortDwells()` (already applied)

The `latestTransitions` variable used by grouped mode is the output of the
`latestSnapshot` memo, which already applied `collapseShortDwells()`.

### Step 2: `preprocess()` (`GraphView.tsx:267-274`)


### Step 3: `buildDirectedGraph()` (`GraphView.tsx:276`)

```typescript

```

Builds a `DirectedGraph` from the preprocessed transitions. See Section 8.

### Step 4: `directedLouvain()` (`GraphView.tsx:277`)

```typescript

```

Runs community detection with the UI-tunable resolution parameter. See Section 9.

### Step 5: Populate nodes and edges from result (`GraphView.tsx:422-477`)

The `useEffect` on `[activeTab, groupedResult, ...]`:

1. Saves old node positions (`GraphView.tsx:428-431`).
2. Computes first/last seen from preprocessed transitions (`GraphView.tsx:433-445`).
3. Clears nodes and edges (`GraphView.tsx:447-448`).
4. Creates nodes from `graph.nodes` with preserved or random positions
   (`GraphView.tsx:450-462`).
5. Creates edges from `graph.edges` using `\0` separator parsing
   (`GraphView.tsx:464-469`).

## 6. Packer Data Pipeline (for contrast)

**Location**: `src/aggregation/packer.ts:72-140`

The packer's `partitionIntoGroups()` function runs:

```
raw transitions → preprocess() → buildDirectedGraph() → directedLouvain() → assignBundles()
```

### Critical difference from graph view

**The packer does NOT call `collapseShortDwells()`**. It passes raw transitions
directly to `preprocess()`. The graph view applies `collapseShortDwells()`
first, then feeds the result to the same `preprocess()` → `buildDirectedGraph()`
→ `directedLouvain()` chain.

| Step | Graph View (Grouped) | Packer |
|------|---------------------|--------|
| 1. `collapseShortDwells()` | YES (`GraphView.tsx:239`) | **NO** |
| 2. `preprocess()` | YES (`GraphView.tsx:267`) | YES (`packer.ts:83`) |
| 3. `buildDirectedGraph()` | YES (`GraphView.tsx:276`) | YES (`packer.ts:100`) |
| 4. `directedLouvain()` | YES (`GraphView.tsx:277`) | YES (`packer.ts:114`) |
| 5. `assignBundles()` | NO (visualization only) | YES (`packer.ts:128`) |
| 6. `graphToEdges()` | NO (reads graph.edges directly) | YES (`packer.ts:137`) |

### Additional packer-only parameter differences

- The packer calls `preprocess()` with **no options** (uses all defaults):
  `preprocess(transitions)` (`packer.ts:83`)
- The graph view passes **UI-tunable options**: resolution, sentinelPassthroughMs,
  sentinelBreakMs, transientDwellMs, hubThresholdPercent, hubMinSources,
  targetPerChunk (`GraphView.tsx:267-274`)
- The packer calls `directedLouvain(graph)` with **no resolution parameter**
  (defaults to 1.0) (`packer.ts:114`)
- The graph view calls `directedLouvain(graph, resolution)` with a
  **tunable resolution** (`GraphView.tsx:277`)

## 7. Shared Preprocessing Pipeline: `preprocess()`

**Location**: `src/aggregation/preprocess.ts:303-320`

```typescript
export function preprocess(
    raw: Transition[],
    options?: PreprocessOptions,
): PreprocessResult {
    const opts = resolveOptions(options);
    const step1 = splitSentinels([...raw], opts);
    const step2 = removeTransients(step1.transitions, opts);
    const hubSources = detectHubs(step2.transitions, opts);
    const step4 = chunkHubs(step2.transitions, hubSources, opts);
    return {
        transitions: step4.transitions,
        excludedSources: step2.excludedSources,
        sentinelCount: step1.sentinelCount,
        chunkMap: step4.chunkMap,
        hubSources,
    };
}
```

### Stage 1: `splitSentinels()` (`preprocess.ts:60-133`)

Handles `off_browser` sentinel transitions. Scans for paired
`A → off_browser` + `off_browser → B` patterns:

- **< sentinelPassthroughMs** (default 2s): Collapse to direct `A → B` edge.
  Filters notification popups, accidental focus loss.
- **sentinelPassthroughMs ≤ x < sentinelBreakMs** (2s-10min): Keep as ephemeral
  node `off_browser:{counter}`. Represents a meaningful but not session-breaking
  absence.
- **≥ sentinelBreakMs** (default 10min): Drop both transitions. Treats long
  absences as task boundaries.
- Unpaired sentinels (`A → off_browser` without return, or `off_browser → B`
  without departure): dropped.

### Stage 2: `removeTransients()` (`preprocess.ts:137-179`)

Removes sources that were visited too briefly to be meaningful:

- **Pass 1 — Dwell detection** (`preprocess.ts:146-160`): If ALL outgoing
  transitions from a source have `dwellMs < transientDwellMs` (default 500ms),
  mark as transient.
- **Pass 2 — Chain scanning** (`preprocess.ts:162-171`): If three consecutive
  transitions each have `dwellMs < transientChainMs` (default 1000ms), mark
  the middle source as transient.
- **Filter** (`preprocess.ts:173-176`): Remove all transitions where `from` or
  `to` is transient. Returns excluded sources in `excludedSources` set.

### Stage 3: `detectHubs()` (`preprocess.ts:183-221`)

Identifies high-degree sources (e.g. email, Slack) that connect to many
other sources:

- Counts unique neighbors (both directions) for each source.
- **Threshold**: `neighbors.size > allSources.size * hubThresholdPercent`
  (default 10%).
- **Guard**: Only activates when `allSources.size >= hubMinSources` (default 15).
- Returns `Set<string>` of hub source IDs.

### Stage 4: `chunkHubs()` (`preprocess.ts:241-299`)

Splits hub sources into time-based chunks so they don't dominate a single
community:

- **Window calculation** (`preprocess.ts:225-239`):
  - `sessionMs = max(ts) - min(ts)` of all hub transitions.
  - `idealChunks = floor(transitionCount / targetPerChunk)` (default target=4).
  - `rawWindowMs = sessionMs / idealChunks`.
  - Clamped to `[minChunkMs, maxChunkMs]` (default [60s, 900s]).
- **Rewriting** (`preprocess.ts:273-296`): Replaces hub source IDs in
  transitions with `hub:{source}:{chunkIndex}`.
- Stores chunk metadata in `chunkMap: Map<string, ChunkInfo>`.

### Default constants (`preprocess.ts:6-15`)

```
SENTINEL_PASSTHROUGH_MS = 2_000
SENTINEL_BREAK_MS       = 600_000
TRANSIENT_DWELL_MS      = 500
TRANSIENT_CHAIN_MS      = 1_000
HUB_THRESHOLD_PERCENT   = 0.1
HUB_MIN_SOURCES         = 15
TARGET_PER_CHUNK         = 4
MIN_CHUNK_MS             = 60_000
MAX_CHUNK_MS             = 900_000
```

## 8. Shared Graph Construction: `buildDirectedGraph()`

**Location**: `src/aggregation/directed-louvain.ts:10-35`

```typescript
export function buildDirectedGraph(transitions: Transition[]): DirectedGraph {
    const nodes = new Set<string>();
    const edges = new Map<string, number>();  // key: "from\0to"
    const inDegree = new Map<string, number>();
    const outDegree = new Map<string, number>();
    let totalWeight = 0;

    for (const t of transitions) {
        nodes.add(t.from);
        nodes.add(t.to);
        const key = t.from + "\0" + t.to;
        edges.set(key, (edges.get(key) ?? 0) + 1);
        outDegree.set(t.from, (outDegree.get(t.from) ?? 0) + 1);
        inDegree.set(t.to, (inDegree.get(t.to) ?? 0) + 1);
        totalWeight++;
    }

    for (const node of nodes) {
        if (!inDegree.has(node)) inDegree.set(node, 0);
        if (!outDegree.has(node)) outDegree.set(node, 0);
    }

    return { nodes, edges, inDegree, outDegree, totalWeight };
}
```

Edge keys use null byte (`\0`) separator. Weights are transition counts.

## 9. Shared Community Detection: `directedLouvain()`

**Location**: `src/aggregation/directed-louvain.ts:54-337`

Directed Louvain algorithm for community detection. Multi-pass, two-phase
per pass:

### Constants (`directed-louvain.ts:3-6`)

```
DEFAULT_RESOLUTION      = 1.0
MIN_IMPROVEMENT         = 1e-6
MAX_PASSES              = 10
MAX_LOCAL_ITERATIONS    = 100
```

### Phase 1: Local greedy moves (`directed-louvain.ts:106-226`)

- Each node starts in its own community.
- Fisher-Yates shuffle to avoid order bias.
- For each node: try moving to each neighbor's community, compute `deltaQ`
  (modularity gain), move to best if positive.
- Repeat up to `MAX_LOCAL_ITERATIONS` until no moves improve modularity.

### Phase 2: Coarsening (`directed-louvain.ts:228-307`)

- Collapse communities into super-nodes.
- Sum edge weights between super-nodes.
- Rebuild adjacency lists for the coarsened graph.
- Continue outer loop until no merges or modularity gain < `MIN_IMPROVEMENT`.

### Output

```typescript
// src/aggregation/types.ts:41-45
type LouvainResult = {
    communities: Map<string, string>;  // nodeId → communityId
    modularity: number;                 // quality metric [0,1]
};
```

## 10. Force-Directed Layout Simulation

**Location**: `GraphView.tsx:480-532` (`tick` callback)

### Physics constants (`GraphView.tsx:62-68`)

```
DEFAULT_CHARGE_K    = 800       (repulsive force multiplier)
DEFAULT_SPRING_K    = 0.02      (spring attractive force)
DEFAULT_REST_LENGTH = 120       (equilibrium edge length in px)
DEFAULT_CENTER_K    = 0.005     (centering force toward origin)
DEFAULT_DAMPING     = 0.85      (velocity decay per tick)
MIN_DIST            = 20        (minimum center-to-center distance)
ENERGY_THRESHOLD    = 0.05      (convergence threshold)
```

### Per-tick forces

1. **Pairwise repulsion** (`GraphView.tsx:485-499`):
   `force = chargeK / dist²`. Applied symmetrically between every pair of nodes.

2. **Spring attraction** (`GraphView.tsx:503-518`):
   For each edge: `force = springK × (dist - restLength)`. Pulls connected
   nodes toward the rest length.

3. **Centering** (`GraphView.tsx:521-523`):
   `force = centerK × position`. Pulls all nodes toward the origin.

4. **Damping + position update** (`GraphView.tsx:524-528`):
   `v *= damping; x += vx; y += vy`.

5. **Convergence check** (`GraphView.tsx:530-531`):
   `totalEnergy = Σ(vx² + vy²)`. If below `ENERGY_THRESHOLD`, the simulation
   sleeps (`awakeRef.current = false`).

### Tunable physics panel (`GraphView.tsx:1647-1734`)

A floating panel (bottom-left) with sliders for all 5 physics parameters.
Reset button restores defaults. All sliders are visible only in graph view
mode.

## 11. Canvas Rendering

**Location**: `GraphView.tsx:534-877` (`draw` callback)

### Rendering order

1. **Background** (`GraphView.tsx:547-550`): Solid `#09090b` fill.

2. **Dot grid** (`GraphView.tsx:563-588`): Adaptive grid of 1.2px dots at
   `DOT_SPACING=24` px intervals. Capped at `MAX_DOTS=5000` to prevent
   performance issues at extreme zoom-out.

3. **Community hulls** (`GraphView.tsx:590-642`, grouped mode only):
   - Collects nodes per community from `louvain.communities`.
   - Computes convex hull using Andrew's monotone chain algorithm
     (`GraphView.tsx:93-130`).
   - Expands hull outward by 20px padding from centroid.
   - Fills with community color at 8% opacity, strokes at 30% opacity.
   - Skipped for communities with < 2 nodes.

4. **Edges** (`GraphView.tsx:644-741`):
   - Parallel edges offset by `EDGE_OFFSET=5` px perpendicular to edge
     direction (prevents A→B and B→A from overlapping).
   - Line width: `1 + log2(weight)` px.
   - Arrowhead: triangle at target node edge, size `4 + log2(weight) × 2`
     (capped at 12px).
   - Edge weight label at midpoint (10px sans-serif).
   - **Focus mode coloring** (when hovering a node with focus mode on):
     - Outbound edges: `rgba(96,165,250,0.6)` (blue)
     - Inbound edges: `rgba(74,222,128,0.6)` (green)
     - Dimmed edges: `rgba(160,160,160,0.1)`
     - Normal: `rgba(160,160,160,0.4)`

5. **Nodes** (`GraphView.tsx:753-855`):
   - Radius: `NODE_RADIUS(8) + min(log2(degree+1) × 2, 8)` px.
   - Degree = sum of edge weights touching the node (`GraphView.tsx:743-751`).
   - **Color logic** (`GraphView.tsx:775-792`):
     - Dimmed (focus mode, not connected): `rgba(120,120,120,0.1)`
     - Grouped mode: community color from `getCommunityColor()`
     - Unknown/off_browser: `rgba(120,120,120,0.4)`
     - Hovered: lightened variant of `nodeColor()`
     - Normal: `nodeColor(source)` — hue-based hash (`GraphView.tsx:163-172`)
   - **Special outlines**:
     - `unknown` nodes: 3px/3px dashed outline
     - `off_browser` nodes: 6px/4px dashed outline
   - **Labels** (below node, `GraphView.tsx:816-854`):
     - Bold 11px context name (before `@`)
     - 9px tab ID (after `@`)
     - 9px URL hostname (blue-tinted)

6. **Empty state** (`GraphView.tsx:857-869`): Centered message when no nodes.

## 16. Summary: Graph View vs Packer Pipeline Comparison

```
GRAPH VIEW (Grouped mode):

    state.snapshot.transitions
        │
        ▼
    collapseShortDwells()          ← GRAPH-VIEW ONLY (5s threshold)
        │
        ▼
    preprocess(opts)               ← with tunable UI parameters
        │
        ▼
    buildDirectedGraph()
        │
        ▼
    directedLouvain(graph, γ)      ← with tunable resolution
        │
        ▼
    [render to canvas]


PACKER (partitionIntoGroups):

    bundler.transitions
        │
        ▼                          ← NO collapseShortDwells()
    preprocess()                   ← with default parameters only
        │
        ▼
    buildDirectedGraph()
        │
        ▼
    directedLouvain(graph)         ← default resolution (1.0)
        │
        ▼
    assignBundles()
        │
        ▼
    [create Packet with Groups]
```

The packer lacks `collapseShortDwells()` and uses hardcoded defaults for all
preprocessing parameters. The graph view applies `collapseShortDwells()` as a
first pass and allows interactive tuning of all parameters via the side panel.
